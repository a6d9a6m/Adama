# adama 项目规划

> 系统规划与任务拆解。

## 项目目标

- 构建商品、订单、用户、任务四类核心服务。
- 支撑秒杀场景下的高并发下单、库存扣减、异步处理与服务治理。
- 持续补齐缓存、消息队列、事务一致性、压测与部署能力。

## 技术课题

- [x] 如何解决超卖
  方案：Redis 预扣库存

- [x] 如何处理订单异步化
  方案：Kafka 消息队列削峰

- [x] 如何实施压力测试
  方案：wrk

- [x] 如何实现对象级缓存
  方案：Redis

- [x] 如何处理分布式事务
  方案：DTM / TCC

- [ ] 如何防刷
  方案：限流、固定时间窗口、用户维度控制

- [ ] 如何防止链接提前泄露
  方案：商品 ID 哈希化、动态路由

- [x] 如何处理缓存与数据库一致性
  方案：异步消息、补偿机制、状态校验

- [x] 如何处理超时未支付订单
  方案：延迟消息 + 订单状态回查 + 库存回补

- [ ] 如何优化秒杀场景 SQL
  方案：热点隔离、异步写入、索引优化

- [ ] 如何实现分库分表
  方案：中间件或业务分片

## 下一阶段优先级

1. 完善 DTM / TCC 事务链路，补齐库存与订单一致性。
2. 完善支付超时关闭与库存回补机制。
3. 继续压测秒杀核心接口，形成稳定的性能基线。
4. 补齐鉴权、网关、监控与部署链路。
