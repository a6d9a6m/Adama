# adama 项目规划

> 系统规划与任务拆解。

## 项目目标

- 构建商品、订单、用户、任务四类核心服务。
- 支撑秒杀场景下的高并发下单、库存扣减、异步处理与服务治理。
- 持续补齐缓存、消息队列、事务一致性、压测与部署能力。

  方案：Redis

- [x] 如何防刷
  方案：Gateway / Nginx 限流、用户维度控制

- [ ] 如何防止链接提前泄露
  方案：商品 ID 哈希化、动态路由

- [x] 如何处理缓存与数据库一致性
  方案：异步消息、补偿机制、状态校验

  方案：延迟消息 + 订单状态回查 + 库存回补

- [x] 如何优化秒杀场景 SQL
  方案：热点锁削减、单 SQL 条件扣减、异步写入

- [ ] 如何实现分库分表
  方案：中间件或业务分片
